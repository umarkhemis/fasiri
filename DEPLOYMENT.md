# Fasiri - Production Deployment Guide

## Choose your deployment path

| Option | Cost | Effort | Best for |
|---|---|---|---|
| **Railway** | Free - $5/mo | 5 minutes | Fastest, no DevOps |
| **Render** | Free - $7/mo | 10 minutes | Free tier with Redis |
| **DigitalOcean VPS** | $6/mo | 30 minutes | Full control, custom domain |
| **Docker (self-hosted)** | VPS cost | 20 minutes | Any Linux server |

---

## Option A - Railway (Recommended - Fastest)

Railway auto-detects the Dockerfile and deploys in minutes.

```bash
# 1. Install Railway CLI
npm install -g @railway/cli

# 2. Login
railway login

# 3. Deploy from your repo root
railway up

# 4. Set environment variables in the Railway dashboard:
#    HUGGINGFACE_API_KEY = hf_...
#    SUNBIRD_API_KEY     = ey...
#    FASIRI_SECRET_KEY = (openssl rand -hex 32)
#    REDIS_URL           = (add a Redis plugin in Railway dashboard)
```

Your API is live at: `https://your-project.up.railway.app`

## Option C - DigitalOcean VPS ($6/month)

### Step 1 - Create a Droplet

- Go to cloud.digitalocean.com → Create Droplet
- **Image:** Ubuntu 22.04 LTS
- **Size:** Basic, $6/month (1 vCPU, 1GB RAM) - enough for Fasiri
- **Region:** Choose closest to your users (Frankfurt for Africa works well)
- **Auth:** SSH key (add your public key)

### Step 2 - Point your domain

In your domain registrar (Namecheap, Cloudflare, etc.):
```
A record:  api.yourdomain.com  →  YOUR_DROPLET_IP
```
Wait 5 minutes for DNS to propagate.

### Step 3 - Run the setup script

SSH into your droplet and run:
```bash
ssh root@YOUR_DROPLET_IP

# Run our automated setup script
curl -sL https://raw.githubusercontent.com/umarkhemis/fasiri/main/scripts/setup_server.sh \
  | DOMAIN=api.yourdomain.com bash
```

### Step 4 - Add your API keys

```bash
nano /opt/fasiri/.env
```

Fill in:
```env
HUGGINGFACE_API_KEY=hf_...
SUNBIRD_API_KEY=ey...
FASIRI_SECRET_KEY=<already generated by setup script>
REDIS_URL=redis://redis:6379/0
BASE_URL=https://api.yourdomain.com
```

### Step 5 - Get SSL certificate

```bash
# Stop any running containers first (Certbot needs port 80)
cd /opt/fasiri && docker compose down

# Get certificate
certbot certonly --standalone -d api.yourdomain.com

# Copy certs
cp /etc/letsencrypt/live/api.yourdomain.com/fullchain.pem /opt/fasiri/nginx/ssl/
cp /etc/letsencrypt/live/api.yourdomain.com/privkey.pem   /opt/fasiri/nginx/ssl/

# Update nginx config with your domain
sed -i 's/YOUR_DOMAIN/api.yourdomain.com/g' /opt/fasiri/nginx/conf.d/fasiri.conf
```

### Step 6 - Start everything

```bash
cd /opt/fasiri

# Build and start all services
docker compose up -d --build

# Check status
docker compose ps
docker compose logs -f api
```

### Step 7 - Verify

```bash
curl https://api.yourdomain.com/health
# {"status":"ok","version":"1.0.0","providers":{...}}

# Run smoke test
python smoke_test.py --url https://api.yourdomain.com
```

## Option D - Any Docker host (generic)

```bash
# 1. Clone the repo on your server
git clone https://github.com/umarkhemis/fasiri.git
cd fasiri

# 2. Create .env
cp env.example .env
nano .env   # fill in your keys

# 3. Build and start
docker compose up -d --build

# 4. Check it's running
docker compose ps
curl http://localhost/health
```

## CI/CD - GitHub Actions (auto-deploy on push)

### Setup

1. Go to your GitHub repo → Settings → Secrets → Actions
2. Add these secrets:

| Secret | Value |
|---|---|
| `DOCKER_USERNAME` | Your Docker Hub username |
| `DOCKER_PASSWORD` | Docker Hub access token |
| `DEPLOY_HOST` | Your server IP |
| `DEPLOY_USER` | SSH user (e.g. `ubuntu`) |
| `DEPLOY_SSH_KEY` | Your private SSH key (full contents) |

3. Push to `main` - the pipeline runs automatically:
   - Tests (52 unit tests)
   - Docker build + push to Docker Hub
   - SSH deploy to your server
   - Health check smoke test

## Monitoring

### View logs

```bash
# All services
docker compose logs -f

# API only
docker compose logs -f api

# Nginx access log
docker compose exec nginx tail -f /var/log/nginx/access.log
```

### Check health

```bash
# Basic health
curl https://api.yourdomain.com/health

# Provider connectivity (requires DEBUG=true)
curl https://api.yourdomain.com/api/v1/debug/providers
```

### Common operations

```bash
# Restart API only (zero Redis downtime)
docker compose restart api

# Pull latest image and redeploy
docker compose pull && docker compose up -d --no-deps api

# View resource usage
docker stats

# Check Redis
docker compose exec redis redis-cli info memory
```

## Refresh Sunbird token (when it expires)

```bash
# On your server
cd /opt/fasiri
python get_sunbird_token.py --env-file .env
docker compose restart api
```

Or set a weekly cron job:
```bash
crontab -e
# Add:
0 2 * * 1 cd /opt/fasiri && python get_sunbird_token.py --email you@email.com --password yourpassword --env-file .env && docker compose restart api
```
